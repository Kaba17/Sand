========================
1) DATABASE (MINIMUM TABLES)
========================
Create/ensure these tables exist:

admins (or users):
- id
- email (unique)
- password_hash
- created_at

claims:
- id (uuid)
- tracking_code (short unique string)
- created_at
- status (enum: new, reviewing, submitted, waiting_response, closed)
- airline
- flight_number
- flight_date
- from_airport
- to_airport
- disruption_type (delay|cancel|denied|missed|baggage)
- delay_minutes (nullable)
- reason_text (nullable)
- user_name
- user_phone
- user_email (nullable)

timeline_events:
- id
- claim_id (fk)
- event_type (created|status_change|note|submitted)
- message
- created_at
- created_by (admin|system)

admin_notes (optional, can be timeline_events with type=note)

========================
2) BACKEND API (EXPRESS/Fastify)
========================
PUBLIC:
POST /api/claims
- Validate required fields
- Generate tracking_code
- Insert claim
- Insert timeline event: "Claim created"
- Return { claimId, trackingCode }

GET /api/claims/track?trackingCode=...&phone=...
- Return claim summary + status + timeline

ADMIN AUTH:
POST /api/admin/login
- Email + password
- Return session/JWT cookie/token
- Add middleware to protect admin routes

ADMIN:
GET /api/admin/claims
- List claims (pagination optional)
GET /api/admin/claims/:id
- Claim details + timeline
PATCH /api/admin/claims/:id/status
- Update status
- Add timeline event: "Status changed to X"
POST /api/admin/claims/:id/note
- Add timeline event type=note
POST /api/admin/claims/:id/submit
- Set status=submitted
- Add timeline event: "Marked as submitted" + store channel/date if you have fields
